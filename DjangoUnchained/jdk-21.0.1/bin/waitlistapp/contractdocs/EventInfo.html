EventInfo
Type Complex

Description

An Event is an occurance. Events may be singlular or composite recurring
events based on schedules or other events.

An Event may be explicitly defined as a complex composite recurring event and
expanded into a set of implicit one-time events for easy consumption.

For example, an explicit recurring Event may be created with a Schedule of
MWF10-11 from September 1 through December 10 and blackout dates of Columbus
and Thanksgiving. This would result in 40 implicit Events for the scheduled
mondays, wednesdays, and fridays in that time period.

A typical provider does not allow creates and updates on Implicit Events
while allowing only explicit events to be created.

Example Types of Events are: 1. non-recurring Event 2. recurring Event 3.
non-recurring composite Event 4. recurring composite Event
