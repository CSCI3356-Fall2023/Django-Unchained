id Unique Id String The system assigned unique id to identify this Entity.
Could be implemented as as sequence number or as a UUID.

Attempts to set this value on creates should result in a ReadOnlyException being thrown

An Id:
1. An id is used when the actual value is unimportant and can therefore be a large hex value for example
2. An id value might be 23b9ca9bd203df902
3. An Id is never intended to be used directly by an end user.
4. Ids are assumed to be of different values in different KS implementations
5. Id values are generated by the service implementation if not supplied
6. Id values are never expected to be used in Configuration or Application code		 Required on updates Read only

principalId Principal Id String The Principal Id of the principal on whose behalf the
authenticated principal is acting. If the authenticated
principal is not acting on behalf of a different user, then
this Id should be the same as the Authenticated Principal Id.

(1) User is authorized to only act on behalf of
itself. Principal Id must equal the Authenticated Principal Id
and the authorization is performed on that Id. If the Principal
Id differs from the Authenticated Principal Id, then the user
is not authorized to perform the requested operation.

(2) User is authorized to act on behalf of another user. The
Principal Id differs from the Authentication Principal
Id. Authorization is checked to see if Authenticated Principal
Id can perform the operation on behalf of Principal Id. Then,
authorization is checked to see if Principal Id can perform the
operation.		     PrincipalInfo  
currentDate Current Date Date The current date in this context.
All operations by the provider should use this method to get the current
date when processing instead of using the system's date.

Normally this simply returns the system's date however it can be "mocked"
for testing purposes. This is done by setting the current date to some date
desired for testing. If set to some past or future date the currentDate
still increments normally within this context. i.e. if you set the
current date to be 12-31-1999 23:59 and wait one minute you can relive
Y2K because getCurrentDate will then return 01-01-2000.

Additionally there are dynamic attributes that also affect this method:

If FREEZE_TIME is set (i.e. freeze.time=true) then whatever was set as the
currentDate is returned, time does not march on, it remains the same value.

If SLOW_DOWN_TIME_BY (i.e. slow.down.time.by=1) forces the system to wait
at least the specified number of milliseconds before returning the time.

If both are specified FREEZE_TIME takes precedence.
