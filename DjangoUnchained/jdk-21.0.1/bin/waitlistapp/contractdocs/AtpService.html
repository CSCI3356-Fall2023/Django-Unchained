Java Package org.kuali.student.r2.core.atp.service

Academic Time Period Service Description and Assumptions. This
service supports the management of Academic Time Periods and their
associated Milestones. The intent is to provide a flexible but
structured way to define the various time frames that are used
throughout the definition, offering and scheduling of Learning
Units.

Method getAtpsByCode
Description Gets a list of ATPs by Code. This method is intended to facilitate a
lookup of ATPs via the partial entry of it's code. It is envisioned for
example to be suitable for a type-ahead lookup. Therefore the
implementation of this method should select all info objects whose code
"starts with" the specified input code up to the length of the input. The
comparison should (generally) ignore the case of the two codes being
compared. This is because most coding schemes do not (and should not)
distinguish between upper and lower case. The consumer should NOT also
not be expected to specify any wildcard characters. Further if they do
specify a "common" wildcard character such as * or % then those
characters should not be treated as a wildcard in the comparison.
Therefore "ENGL" would match "ENGLISH" but "ENGL%" should not match
"ENGLISH". Specifying an empty string should return an empty list. Some
implementations for performance reasons may also return an empty list for
one or two or some other minimum number of characters. The implementation
may also decide to return a smaller limited set of items if the actual
number of matches is deemed too large. If the implementation does limit
the results it should always return all the objects who's code fully
matches the input text. This is to make sure this method can also be used
to reliably fetch an object by it's full code.
Parameters String code an ATP Code
ContextInfo contextInfo information containing the principalId and
locale information about the caller of service operation
Return AtpInfoList a list of ATPs with the given ATP Code
Errors InvalidParameterException contextInfo is not valid
MissingParameterException code or contextInfo is
missing or null
OperationFailedException unable to complete request
PermissionDeniedException an authorization failure
occurred

Description Validates an Academic Time Period. Depending on the value of
validationType, this validation could be limited to tests on
just the current object and its directly contained sub-objects
or expanded to perform all tests related to this ATP. If an
identifier is present for the ATP (and/or one of its contained
sub-objects) and a record is found for that identifier, the
validation checks if the ATP can be shifted to the new
values. If a an identifier is not present or a record does not
exist, the validation checks if the ATP with the given data can
be created.
