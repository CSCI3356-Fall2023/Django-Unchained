Description

All changes to a student registration is performed by submitting a
RegistrationRequest. This request represents an overall transaction among a
set of RegistrationRequestItems (register, add, drop, update, etc.)

For every transactional operation from the application, a new Request is
created.

The abstract Request entity is a Relationship to its canonical counterpart.
Taking a shortcut here and not incuding the canonical, but surfacing the
effective dates to zoom in on effective requests. An expired request is one
that has been processed or canceled. This may be shortsighted but canonical
requests is something new to the pattern.
